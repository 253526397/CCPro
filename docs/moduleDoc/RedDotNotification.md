# 红点提示模块

## 概述

一个游戏内通用的红点提示模块，支持树状结构管理和数据-视图分离。

---

## 核心要求

### 1. 数据层与视图层分离

| 层 | 职责 |
|---|---|
| **数据逻辑层** | 负责红点数据的计数、聚合、状态管理 |
| **视图逻辑层** | 负责红点 UI 的显示/隐藏、样式控制 |

数据层不直接操作 UI，视图层通过监听数据层变化来更新显示。

### 2. 树状路径管理

红点节点以 **路径形式** 组织为树状结构，示例：

```
home/friends              → 主界面好友入口的红点
home/friends/message      → 好友未读消息的红点
home/friends/apply        → 好友申请的红点
```

每个 `/` 分隔符代表一层父子关系。路径天然定义了节点的层级，便于理解和维护。

### 3. 计数聚合规则

- 每个节点可拥有**自身的计数值**
- **父节点的计数值** = 自身计数 + 所有子节点的计数之和
- 计数为 `0` 时自动隐藏红点，`> 0` 时自动显示

```
示例：
  home/friends          count = 0 (自身) + 5 (message) + 2 (apply) = 7
  home/friends/message  count = 5
  home/friends/apply    count = 2
```

### 4. CocosCreator 组件脚本

提供一个 CocosCreator 脚本组件，在组件生命周期中：

- **`onLoad` / `start`**：通过路径向红点模块注册当前显示对象
- **`onDestroy`**：从红点模块移除当前显示对象
- 红点模块根据路径对应的计数值，**统一控制**红点的显示或隐藏

> **重要**：同一个节点路径下可以同时注册**多个显示对象**。例如路径 `home/friends` 可以挂载在主界面的好友入口按钮上，同时也可以挂载在侧边栏的好友图标上 —— 路径计数变化时，所有关联的显示对象都会同步更新。

```typescript
// 在 CocosCreator 组件中挂载红点脚本
// 注册路径为 "home/friends/message"
// 数据层更新计数时，视图自动响应显示/隐藏
```
